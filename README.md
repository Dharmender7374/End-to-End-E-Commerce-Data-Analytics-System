# End-to-End-E-Commerce-Data-Analytics-System

# 📌 Overview

This project demonstrates a complete end-to-end data analytics workflow using a real-world e-commerce dataset.
It covers the full pipeline from raw data ingestion and transformation in PostgreSQL to building interactive dashboards in Power BI for business insights.

The goal is to simulate how data analysts design scalable reporting systems and extract meaningful insights from large datasets.

# 🛠️ Tech Stack

1. PostgreSQL
2. Power BI

# 📂 Dataset

A. Source: Kaggle
B. Dataset includes:

  1. Customers
  2. Orders
  3. Order Items
  4. Payments
  5. Reviews
  6. Products
  7. Sellers
  8. Geolocation
  9. Category Translation

# ⚙️ Project Workflow

1. Understanding the structure of a real-world e-commerce dataset
2. Setting up PostgreSQL database for analytics workflows
3. Creating tables and importing large CSV datasets
4. Data validation, cleaning, and SQL best practices
5. Optimizing queries using indexes
6. Creating analytical SQL views for BI reporting
7. Connecting PostgreSQL with Power BI using DirectQuery
8. Building business metrics and KPIs
9. Designing a complete analytics workflow from database to insights
